#### キャパシティ計算式

```
必要RCU = (読み取り回数/秒) × ceil(アイテムサイズ/4KB) × 整合性係数
必要WCU = (書き込み回数/秒) × ceil(アイテムサイズ/1KB)

※整合性係数: 強い整合性=1, 結果整合性=0.5, トランザクション=2
```

#### ScalarDB使用時のオーバーヘッド

ScalarDBはトランザクション管理のため追加のI/Oが発生します。

```
ScalarDBストレージオーバーヘッド:
- Afterイメージ: +60 bytes/レコード（トランザクションメタデータ）
- Beforeイメージ: +60 bytes/レコード + 非PKカラムのコピー
- 合計: 120 bytes + 非PKカラムサイズ

例: 1KBアイテム → ScalarDB経由で約2KB（2倍）
```

```
ScalarDB I/Oオーバーヘッド（1レコード更新あたり）:
- 読み取り(Before取得): ceil(2KB/4KB) = 1 RCU（強整合性）
- 書き込み(After+Before): ceil(2KB/1KB) = 2 WCU
- Coordinator書き込み: 1 WCU
- 合計: 1 RCU + 3 WCU（元の1 WCUと比較して約3-5倍）
```

#### プロビジョンドモード月額計算

```
月額(USD) = (WCU × $0.00065 + RCU × $0.00013) × 730時間
月額(USD) = WCU × $0.4745 + RCU × $0.0949

例: 10,000 WCU + 10,000 RCU
= $4,745 + $949 = $5,694/月
```

#### Request Unit（RU）計算式

```
月額(USD) = (プロビジョンドRU/s × $0.00008/RU/時) × 730時間
月額(USD) ≈ RU/s × $0.0584

例: 10,000 RU/s = $584/月 ≈ $600/月
```

#### RU消費量の目安

| 操作 | 消費RU（1KBアイテム） |
|-----|---------------------|
| ポイント読み取り（1項目） | 1 RU |
| 書き込み | 5 RU |
| クエリ（単純） | 2〜3 RU |
| クエリ（複雑・スキャン） | 10〜100+ RU |

#### ScalarDB使用時のオーバーヘッド

```
ScalarDB経由でのRU消費（1レコード更新あたり）:
- 読み取り(Before取得): 1 RU
- 書き込み(After+Before): 10 RU（5 RU × 2倍サイズ）
- Coordinatorテーブル書き込み: 5 RU
- 合計: 約16 RU（元の5 RUと比較して約3倍）
```
